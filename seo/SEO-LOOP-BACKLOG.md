# Street Teams Co Daily SEO Ship Loop

One SEO play per day, gated by `npm run build`, pushed to `redesign-v5` (which also deploys to `main`/prod).

## Hard rules (binding, do not violate)

- This is a Vite/React SPA. Any NEW route MUST get a prerendered HTML page (see `scripts/generate-static.mjs` and `scripts/generators/*.mjs`), or crawlers only ever see the homepage. Re-run the sitemap generator (`generate-sitemap.cjs`) after adding routes — it already runs as part of `npm run build`.
- ABSOLUTELY NO PRICING OR RATES anywhere, including JSON-LD structured data. `/pricing` intentionally redirects to `/contact`. The business is custom-quote only. Never add a pricing page or dollar amounts.
- CONTENT SATURATION: before adding any new guide/blog page, grep `public/blog/` and existing routes for overlapping topics. This site already has 170+ blog posts and dozens of near-duplicate topic pages (cost/pricing guides, city guides, "how to choose" guides, etc.). Prefer improving existing pages over adding new ones.
- Never invent stats, client names, reviews, or case studies. Only reuse claims already present in the site copy (e.g. "500+ campaigns", "94% client retention", "1,000+ cities").
- No em dashes in site copy; use commas or periods.
- Additive changes only; never delete pages.
- Gate every change with `npm run build` (installs may be needed first: `npm install`) and require success before pushing.
- Stage only the files you touched, never `git add -A`.

## Important structural note (discovered on first run)

Meta title/description for key pages exist in **three separate places** that are NOT synced with each other:
1. `src/pages/*.tsx` (`useMetaTags` calls) — only takes effect client-side after JS hydration.
2. `scripts/generators/*.mjs` (`standalone-pages.mjs`, `service-pages.mjs`, etc.) — generates the actual prerendered static HTML under `dist/` that crawlers and social scrapers see. **This is the source of truth for SEO.**
3. Root `index.html` — hardcoded shell meta tags used for the `/` route specifically.

When auditing or changing title/meta/schema for any page, check all three locations, not just the React component.

## FAQ JSON-LD audit note (play 3 run, 2026-07-23)

Before shipping play 3, audited the whole site (home, all 9 `/services/*` pages + index, pricing, testimonials, our-team, all city/state pages, all 172 blog posts, `/compare`, `fifa-world-cup-2026-staffing`, `los-angeles-2028-staffing`) for pages with visible FAQ content but no `FAQPage` JSON-LD. Result: FAQ schema is already implemented essentially everywhere via `scripts/generators/shared.mjs`'s `faqSchema()` helper. Only one real gap found: `public/partners/agencies.html` had a visible "Frequently Asked Questions from Agency Partners" section (5 Q&As, hand-authored static HTML, not run through the generator scripts) with no matching schema. Added a `FAQPage` JSON-LD block using its existing Q&A copy verbatim, no new claims. `public/partners/index.html` has no FAQ content of its own (just links out), so nothing to add there.

**Bigger find while auditing**: `public/faq.html` (the sitewide FAQ page, linked from the global footer on every page via `src/App.tsx` and `scripts/generators/shared.mjs`, and listed in `sitemap.xml`) had explicit dollar-figure rate benchmarks live in production, in both visible copy and its `FAQPage` JSON-LD: "agency bill rates... typically run $25-$75/hour," a BLS citation of "$18/hour (~$37,960/year)," and a question headlined "Why are some agencies charging $200+/hr?" This is exactly the "leaked rate cards through invisible JSON-LD" failure mode the hard rules warn about, already live and indexed. Treated this as a binding-rule compliance fix (not a new feature) and shipped it same-day alongside the play 3 schema addition: rewrote the "How much does a street team marketing campaign cost?" answer (both the visible `<p>` and the JSON-LD `text` field) to drop all dollar figures while keeping the "custom-quoted, no published rate cards, get a free quote" substance, and retitled/rewrote "Why are some agencies charging $200+/hr?" to "Why do agency quotes vary so much?" removing the dollar figure from both heading and body while keeping the markup/specialized-roles explanation. Left the `$5M general liability / $1M professional liability` insurance-coverage figures and the payment-terms language (50% deposit, Net-30, 3% card fee) untouched, those are business insurance/payment-logistics figures, not service pricing/rate cards, and matching figures are already published elsewhere on the site (e.g. `public/partners/agencies.html` stats strip).

Did **not** touch the 4 pricing-guide blog posts flagged in the play 2 note above (`brand-ambassador-pricing-explained.html` etc.) since those are still awaiting an operator decision on whether to keep/rewrite that content; `/faq.html` was different because it is a general FAQ page (not a dedicated pricing-guide article) where dropping a couple of sentences of dollar figures doesn't undercut the page's purpose.
